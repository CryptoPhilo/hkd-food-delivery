# 배달원 정산 시스템 - 상세 설계 문서

## 1. 개요

### 1.1 목적
배달원의 배달 수수료를 체계적으로 관리하고, 월별 정산을 통해 정확한 보수를 지급하는 시스템

### 1.2 범위
- 배달 완료 시 건별 배달비 누적
- 월별 정산 자동 생성
- 수수료/세금 자동 계산
- 정산 승인/지급 플로우
- 배달원별 정산 내역 조회

## 2. 데이터 모델 설계

### 2.1 Settlement (정산) 모델

```prisma
model Settlement {
  id              String   @id @default(uuid())
  driverId        String
  driver          Driver   @relation(fields: [driverId], references: [id])
  
  period          String   // 정산 기간 (형식: "YYYY-MM", 예: "2026-03")
  startDate       DateTime // 정산 시작일
  endDate         DateTime // 정산 종료일
  
  // 배달 실적
  totalDeliveries Int      // 총 배달 횟수
  totalDeliveryFee Int     // 총 배달비 (청구 금액)
  
  // 공제 항목
  serviceFeeRate  Float    // 서비스 수수료율 (예: 0.10 = 10%)
  serviceFee      Int      // 서비스 수수료 금액
  taxRate         Float    // 세율 (예: 0.033 = 3.3%)
  tax             Int      // 세금 (사업소득세)
  
  // 추가/차감
  bonus           Int      @default(0) // 보너스/인센티브
  penalty         Int      @default(0) // 벌금/차감
  
  // 최종 금액
  grossAmount     Int      // 총 금액 (totalDeliveryFee + bonus)
  netAmount       Int      // 실지급액 (grossAmount - serviceFee - tax - penalty)
  
  // 상태 관리
  status          String   @default("pending") 
  // pending: 정산 생성됨
  // calculated: 금액 계산 완료
  // approved: 승인됨
  // paid: 지급 완료
  
  // 지급 정보
  paidAt          DateTime?
  paidAmount      Int?     // 실제 지급액
  bankName        String?  // 은행명
  bankAccount     String?  // 계좌번호
  accountHolder   String?  // 예금주
  
  // 메타 정보
  notes           String?  // 메모
  createdBy       String?  // 생성자 (어드민 ID)
  approvedBy      String?  // 승인자
  approvedAt      DateTime?
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  
  @@unique([driverId, period]) // 동일 기간 중복 정산 방지
  @@index([driverId])
  @@index([period])
  @@index([status])
  @@map("settlements")
}
```

## 4. 비즈니스 로직

### 4.1 정산 생성 로직

```typescript
interface SettlementConfig {
  serviceFeeRate: number;  // 기본: 0.10 (10%)
  taxRate: number;         // 기본: 0.033 (3.3%)
}

interface SettlementData {
  driverId: string;
  period: string;
  startDate: Date;
  endDate: Date;
  totalDeliveries: number;
  totalDeliveryFee: number;
  serviceFee: number;
  tax: number;
  bonus: number;
  penalty: number;
  grossAmount: number;
  netAmount: number;
}

async function generateMonthlySettlement(
  period: string,  // "YYYY-MM"
  config: SettlementConfig = { serviceFeeRate: 0.10, taxRate: 0.033 }
): Promise<SettlementData[]> {
  // 1. 기간 계산
  const [year, month] = period.split('-').map(Number);
  const startDate = new Date(year, month - 1, 1);
  const endDate = new Date(year, month, 0, 23, 59, 59, 999);
  
  // 2. 전체 배달원 조회 (해당 기간에 배달 완료한 배달원)
  const drivers = await prisma.driver.findMany({
    where: {
      orders: {
        some: {
          status: 'completed',
          deliveredAt: {
            gte: startDate,
            lte: endDate
          }
        }
      }
    }
  });
  
  const settlements: SettlementData[] = [];
  
  // 3. 배달원별 정산 계산
  for (const driver of drivers) {
    // 배달 완료 주문 조회
    const orders = await prisma.order.findMany({
      where: {
        driverId: driver.id,
        status: 'completed',
        deliveredAt: {
          gte: startDate,
          lte: endDate
        }
      }
    });
    
    const totalDeliveries = orders.length;
    const totalDeliveryFee = orders.reduce((sum, o) => sum + o.deliveryFee, 0);
    
    // 4. 금액 계산
    const serviceFee = Math.floor(totalDeliveryFee * config.serviceFeeRate);
    const tax = Math.floor((totalDeliveryFee - serviceFee) * config.taxRate);
    const grossAmount = totalDeliveryFee;
    const netAmount = grossAmount - serviceFee - tax;
    
    settlements.push({
      driverId: driver.id,
      period,
      startDate,
      endDate,
      totalDeliveries,
      totalDeliveryFee,
      serviceFee,
      tax,
      bonus: 0,
      penalty: 0,
      grossAmount,
      netAmount
    });
  }
  
  return settlements;
}
```

### 4.2 정산 승인 로직

```typescript
async function approveSettlement(
  settlementId: string,
  adminId: string,
  notes?: string
): Promise<Settlement> {
  // 1. 정산 조회
  const settlement = await prisma.settlement.findUnique({
    where: { id: settlementId }
  });
  
  if (!settlement) {
    throw new Error('정산을 찾을 수 없습니다');
  }
  
  // 2. 상태 검증
  if (settlement.status !== 'calculated') {
    throw new Error('계산 완료 상태의 정산만 승인할 수 있습니다');
  }
  
  // 3. 승인 처리
  const updated = await prisma.settlement.update({
    where: { id: settlementId },
    data: {
      status: 'approved',
      approvedBy: adminId,
      approvedAt: new Date(),
      notes
    }
  });
  
  return updated;
}
```

### 4.3 지급 완료 로직

```typescript
async function markSettlementAsPaid(
  settlementId: string,
  paidAmount: number,
  paidAt: Date,
  notes?: string
): Promise<Settlement> {
  // 1. 정산 조회
  const settlement = await prisma.settlement.findUnique({
    where: { id: settlementId }
  });
  
  if (!settlement) {
    throw new Error('정산을 찾을 수 없습니다');
  }
  
  // 2. 상태 검증
  if (settlement.status !== 'approved') {
    throw new Error('승인된 정산만 지급 처리할 수 있습니다');
  }
  
  // 3. 금액 검증 (선택적)
  if (paidAmount !== settlement.netAmount) {
    // 차이가 있는 경우 경고 로그
    console.warn(`정산 금액 불일치: 예상 ${settlement.netAmount}, 실제 ${paidAmount}`);
  }
  
  // 4. 지급 처리
  const updated = await prisma.settlement.update({
    where: { id: settlementId },
    data: {
      status: 'paid',
      paidAmount,
      paidAt,
      notes
    }
  });
  
  return updated;
}
```

### 4.4 예상 정산 계산 로직

```typescript
async function calculateCurrentSettlement(
  driverId: string
): Promise<CurrentSettlement> {
  const now = new Date();
  const year = now.getFullYear();
  const month = now.getMonth();
  
  const startDate = new Date(year, month, 1);
  const endDate = now;
  
  // 이번 달 배달 완료 주문 조회
  const orders = await prisma.order.findMany({
    where: {
      driverId,
      status: 'completed',
      deliveredAt: {
        gte: startDate,
        lte: endDate
      }
    }
  });
  
  const totalDeliveries = orders.length;
  const totalDeliveryFee = orders.reduce((sum, o) => sum + o.deliveryFee, 0);
  
  const serviceFeeRate = 0.10;
  const taxRate = 0.033;
  
  const estimatedServiceFee = Math.floor(totalDeliveryFee * serviceFeeRate);
  const estimatedTax = Math.floor((totalDeliveryFee - estimatedServiceFee) * taxRate);
  const estimatedNetAmount = totalDeliveryFee - estimatedServiceFee - estimatedTax;
  
  const daysInMonth = new Date(year, month + 1, 0).getDate();
  const daysRemaining = daysInMonth - now.getDate();
  
  return {
    period: `${year}-${String(month + 1).padStart(2, '0')}`,
    toDate: now.toISOString().split('T')[0],
    totalDeliveries,
    totalDeliveryFee,
    estimatedServiceFee,
    estimatedTax,
    estimatedNetAmount,
    daysRemaining
  };
}
```

## 5. 상태 관리

### 5.1 정산 상태 전이

```
pending → calculated → approved → paid
   │          │           │
   └──────────┴───────────┴──→ cancelled (에러 시)
```

| 현재 상태 | 다음 상태 | 조건 |
|-----------|-----------|------|
| pending | calculated | 금액 계산 완료 |
| calculated | approved | 어드민 승인 |
| approved | paid | 지급 완료 |
| any | cancelled | 오류/취소 |

### 5.2 상태 변경 규칙

- **pending → calculated**: 자동 (정산 생성 시)
- **calculated → approved**: 어드민 수동 승인
- **approved → paid**: 어드민 수동 지급 처리
- **역방향 이동**: 불가 (취소만 가능)

## 6. 검증 규칙

### 6.1 정산 생성 시 검증

- [ ] 기간 형식 검증 (YYYY-MM)
- [ ] 중복 정산 방지 (동일 배달원, 동일 기간)
- [ ] 배달 내역 존재 여부
- [ ] 수수료율 범위 검증 (0~1)

### 6.2 승인 시 검증

- [ ] 상태가 'calculated'인지 확인
- [ ] 금액 재계산 후 일치 여부 확인
- [ ] 어드민 권한 확인

### 6.3 지급 시 검증

- [ ] 상태가 'approved'인지 확인
- [ ] 지급액이 0보다 큰지 확인
- [ ] 지급일이 유효한지 확인
- [ ] 계좌 정보 존재 여부 확인
