# 한경 음식배달 서비스 - 시스템 설계 문서 (v2.0)

## 1. 개요

### 1.1 프로젝트 목적
제주 한경면 지역의 음식 배달 서비스 플랫폼으로, 고객-식당-배달원을 연결하는 통합 시스템

### 1.2 확장 범위
- **기존**: 음식 배달 (식당 메뉴)
- **확장**: 편의점 상품 배달 + 성인 인증 (주류 등)

### 1.3 기술 스택

| 구분 | 기술 | 버전 | 비고 |
|------|------|------|------|
| Frontend | Next.js | 14.2.3 | App Router |
| Backend | Express | 4.18.2 | Node.js |
| Database | SQLite/PostgreSQL | - | 개발: SQLite, 운영: PostgreSQL |
| ORM | Prisma | 5.10.0 | - |
| SMS | 알리고 (Aligo) | - | SMS 발송 |
| Payment | PortOne (포트원) | - | 카드 결제 |
| Maps | Naver/Kakao Maps | - | 좌표 변환 |

### 3.2 주요 모델

#### Restaurant (식당/편의점)
- `storeType`: 'restaurant' | 'convenience_store'
- `brandName`: 브랜드명 (CU, GS25 등)
- `operatingHours24`: 24시간 영업 여부

#### Menu (메뉴/상품)
- `requiresAgeVerification`: 성인 인증 필요 여부
- `ageRestriction`: 'none' | 'teen' | 'adult'
- `category`: 카테고리 (식품, 음료, 주류 등)
- `stock`: 재고 수량
- `barcode`: 바코드

#### AgeVerification (성인 인증)
- 사용자별 성인 인증 기록
- 24시간 유효

## 5. 비즈니스 로직

### 5.1 주문 플로우

```
1. 사용자가 식당/편의점 선택
2. 메뉴/상품 선택 → 장바구니
3. 주문 사전 검증 (POST /orders/validate)
   - 주류/음식 비율 검증
   - 성인 인증 필요 여부
4. 결제 (PortOne)
5. 주문 생성 (POST /orders)
   - 재고 확인
   - 배달 가능 지역 확인
   - 성인 인증 확인
6. 주문 확정 (PUT /orders/:id/confirm)
7. 배달원 배정
8. 픽업 → 배달 → 완료
```

### 5.2 성인 인증 플로우

```
1. 사용자가 성인 인증 요청
2. SMS 인증번호 발송 (6자리, 3분 유효)
3. 인증번호 입력
4. 인증 완료 (24시간 유효)
5. 주문 시 ageVerificationId 전달
```

### 5.3 주류/음식 비율 검증

```
규칙: 주류 총액 ≤ 음식 총액
- 주류만 주문: 통과
- 음식만 주문: 통과
- 주류 < 음식: 통과
- 주류 = 음식: 통과
- 주류 > 음식: 실패 (경고 메시지)
```
