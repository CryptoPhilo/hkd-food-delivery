# 한경 음식배달 서비스 - 어드민 업무 흐름 테스트 시나리오

## 1. 시나리오 개요

### 1.1 테스트 대상
- **역할**: 시스템 관리자 (Admin)
- **목적**: 시스템 운영, 식당/주문 관리, 통계 확인

### 1.2 사전 조건
- 어드민 계정으로 로그인
- 관리자 권한 설정 완료

---

## 2. 시나리오 1: 일일 대시보드 확인

### 2.1 시나리오 설명
매일 아침 대시보드를 확인하여 전일 주문 현황과 오늘 예상 실적을 파악

### 2.2 테스트 단계

#### Step 1: 어드민 페이지 접속
| 항목 | 내용 |
|------|------|
| **동작** | 어드민 대시보드 페이지 로드 |
| **페이지** | `/admin` |
| **API** | `GET /api/v1/admin/dashboard` |
| **기대결과** | 주요 지표 표시 |

#### Step 2: 주요 지표 확인
| 항목 | 내용 |
|------|------|
| **확인항목** | 오늘 주문 수, 매출, 평균 주문 금액 |
| **확인항목** | 대기 중인 주문 수 |
| **확인항목** | 배달 완료율 |
| **확인항목** | 취소율 |

#### Step 3: 시간대별 추이 확인
| 항목 | 내용 |
|------|------|
| **동작** | 시간대별 주문 그래프 확인 |
| **기대결과** | 피크 시간대 파악 |
| **활용** | 배달원 스케줄링 참고 |

## 3. 시나리오 2: 주문 관리

### 3.1 시나리오 설명
실시간 주문 모니터링 및 이상 주문 처리

### 3.2 테스트 단계

#### Step 1: 주문 목록 조회
| 항목 | 내용 |
|------|------|
| **동작** | 전체 주문 목록 조회 |
| **페이지** | `/admin/orders` |
| **API** | `GET /api/v1/admin/orders` |
| **기대결과** | 주문 목록 표시 (상태별 필터링) |

#### Step 2: 대기 주문 확인
| 항목 | 내용 |
|------|------|
| **필터** | `status: pending, order_confirmed` |
| **확인항목** | 결제 완료 but 미확정 주문 |
| **조치** | 고객에게 확정 재촉 SMS 발송 |

#### Step 3: 지연 주문 확인
| 항목 | 내용 |
|------|------|
| **필터** | 예상 시간 초과 주문 |
| **확인항목** | preparing 상태로 장시간 방치 |
| **조치** | 식당/배달원에게 확인 전화 |

#### Step 4: 취소 요청 처리
| 항목 | 내용 |
|------|------|
| **동작** | 고객 취소 요청 확인 |
| **API** | `PUT /api/v1/admin/orders/:id/status` |
| **기대결과** | 취소 처리, 환불 진행 |
| **검증항목** | 취소 사유, 환불 상태 |

#### Step 5: 문제 주문 처리
| 항목 | 내용 |
|------|------|
| **동작** | 분쟁 주문 수동 처리 |
| **기대결과** | 상태 변경, 메모 저장 |
| **활용** | 고객 CS 대응 |
