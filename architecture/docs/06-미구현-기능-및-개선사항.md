## 2. 배달원 보수 정산 시스템

### 2.1 현재 상태
- `totalDeliveries`: 총 배달 횟수 누적
- `totalDeliveryFee`: 총 배달비 누적
- **미구현**: 정산 주기, 세금, 수수료, 실지급액 계산

### 2.2 필요 기능

#### 정산 데이터 모델
```prisma
model Settlement {
  id              String   @id @default(uuid())
  driverId        String
  period          String   // "2026-03" (월 정산)
  totalDeliveries Int
  totalDeliveryFee Int
  serviceFee      Int      // 플랫폼 수수료 (예: 10%)
  tax             Int      // 세금 (소득세 등)
  bonus           Int      // 보너스/인센티브
  penalty         Int      // 벌금/차감
  netAmount       Int      // 실지급액
  status          String   // pending, approved, paid
  paidAt          DateTime?
  bankAccount     String   // 계좌번호
  bankName        String   // 은행명
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}
```

#### 정산 API 필요
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | /api/v1/settlements/driver/:driverId | 배달원별 정산 내역 |
| POST | /api/v1/settlements/generate | 월 정산 생성 (어드민) |
| PUT | /api/v1/settlements/:id/approve | 정산 승인 |
| PUT | /api/v1/settlements/:id/pay | 지급 완료 처리 |
| GET | /api/v1/settlements/summary | 전체 정산 요약 |

#### 정산 로직
```typescript
interface SettlementCalculation {
  grossAmount: number;      // 총 배달비
  serviceFeeRate: number;   // 수수료율 (예: 0.1)
  serviceFee: number;       // 수수료
  taxRate: number;          // 세율 (예: 0.033)
  tax: number;              // 세금
  bonus: number;            // 보너스
  penalty: number;          // 벌금
  netAmount: number;        // 실지급액
}

function calculateSettlement(deliveryFee: number): SettlementCalculation {
  const serviceFeeRate = 0.10; // 10%
  const taxRate = 0.033;       // 3.3% (사업소득세)
  
  const serviceFee = Math.floor(deliveryFee * serviceFeeRate);
  const tax = Math.floor((deliveryFee - serviceFee) * taxRate);
  const netAmount = deliveryFee - serviceFee - tax;
  
  return {
    grossAmount: deliveryFee,
    serviceFeeRate,
    serviceFee,
    taxRate,
    tax,
    bonus: 0,
    penalty: 0,
    netAmount
  };
}
```

### 2.3 정산 대시보드 필요
- **배달원용**: 내 정산 내역, 예상 수입, 지급 일정
- **어드민용**: 전체 정산 현황, 미지급 건, 정산 보고서

### 4.2 필요 기능

#### WebSocket 이벤트
| 이벤트 | 발신자 | 수신자 | 데이터 |
|--------|--------|--------|--------|
| order:created | 시스템 | 배달원 | 새 주문 정보 |
| order:assigned | 배달원 | 고객 | 배달원 정보 |
| order:status | 시스템 | 고객 | 상태 변경 |
| order:location | 배달원 | 고객 | 실시간 위치 |
| driver:status | 배달원 | 시스템 | 업무 상태 |

#### 구현 예시
```typescript
// Socket.io 서버 설정
io.on('connection', (socket) => {
  // 배달원 위치 업데이트
  socket.on('driver:location', (data) => {
    const { driverId, lat, lng, orderId } = data;
    // 고객에게 전달
    io.to(`order:${orderId}`).emit('driver:location', {
      driverId,
      lat,
      lng,
      updatedAt: new Date()
    });
  });
  
  // 주문 상태 변경
  socket.on('order:status', (data) => {
    const { orderId, status } = data;
    io.to(`order:${orderId}`).emit('order:status', {
      orderId,
      status,
      updatedAt: new Date()
    });
  });
});
```

#### 캐싱
```typescript
// Redis 캐싱 예시
const CACHE_TTL = 300; // 5분

async function getRestaurants(lat: number, lng: number) {
  const cacheKey = `restaurants:${lat}:${lng}`;
  const cached = await redis.get(cacheKey);
  
  if (cached) {
    return JSON.parse(cached);
  }
  
  const restaurants = await prisma.restaurant.findMany();
  await redis.setex(cacheKey, CACHE_TTL, JSON.stringify(restaurants));
  
  return restaurants;
}
```

## 12. 결론

현재 시스템은 MVP 수준의 핵심 기능이 구현되어 있으나, **보안**, **정산**, **실시간 기능** 등 운영에 필요한 핵심 기능이 미구현 상태입니다.

**즉시 구현 필요**:
1. 어드민 인증/인가 (보안)
2. 입력값 검증 강화 (보안)
3. 결제 보안 (보안)
4. 배달원 정산 시스템 (비즈니스)

**단기 구현 필요** (1~2개월):
1. 푸시 알림
2. 실시간 추적
3. 리뷰 시스템

**중장기 구현** (3~6개월):
1. 프로모션/쿠폰
2. 통계/분석 고도화
3. 성능 최적화
4. 확장 기능
