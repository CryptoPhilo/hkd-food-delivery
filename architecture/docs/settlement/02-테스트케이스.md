# 배달원 정산 시스템 - 테스트 케이스

## 1. 테스트 케이스 개요

### 1.1 테스트 범위
- 정산 생성 (자동/수동)
- 정산 금액 계산 정확성
- 정산 승인 플로우
- 지급 처리 플로우
- 예외 상황 처리

### 1.2 테스트 환경
- DB: SQLite (테스트용)
- 프레임워크: Jest + Supertest
- 격리: 각 테스트 전후 데이터 초기화

| 항목 | 내용 |
|------|------|
| **테스트 ID** | TC-SET-001 |
| **테스트명** | 월 정산 정상 생성 |
| **우선순위** | 높음 |
| **전제조건** | 2026-03 기간에 배달 완료된 주문 150건 존재 (배달원 1명) |
| **테스트步骤** | 1. POST /api/v1/settlements/generate 호출<br>2. period: "2026-03" 전달 |
| **예상결과** | 200 응답, 정산 1건 생성, status: "calculated" |
| **검증항목** | - totalDeliveries = 150<br>- totalDeliveryFee 정확<br>- serviceFee = totalDeliveryFee * 0.10<br>- tax = (totalDeliveryFee - serviceFee) * 0.033<br>- netAmount = totalDeliveryFee - serviceFee - tax |
