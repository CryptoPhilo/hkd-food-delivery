# 한경 배달 서비스 - 편의점 확장 설계 문서

## 1. 개요

### 1.1 목적
기존 음식 배달 서비스에 편의점 상품 배달 기능을 추가하여 배달 품목을 확장합니다.
- 어드민에서 '편의점' 상점 생성 및 상품 등록
- 주류 등 성인 인증 필요 상품에 대한 연령 확인
- 기존 Restaurant 모델 확장 방식으로 빠른 구현

### 1.2 확장 범위
| 항목 | 현재 | 확장 후 |
|------|------|---------|
| 상점 유형 | 식당만 | 식당 + 편의점 |
| 상품 유형 | 메뉴 | 메뉴 + 편의점 상품 |
| 연령 제한 | 없음 | 성인 인증 필요 상품 |
| 주문 검증 | 재고 확인만 | 재고 + 성인 인증 |

### 1.3 구현 상태
✅ **완료됨:**
- Prisma 스키마 확장 (storeType, requiresAgeVerification, AgeVerification)
- StoreService 구현 (편의점 CRUD, 상품 관리)
- AgeVerificationService 구현 (성인 인증 요청/검증)
- Admin Routes 확장 (편의점/상품 관리 API)
- OrderService 확장 (성인 인증 검증, 재고 관리)
- 테스트 코드 작성 (59개 테스트 중 52개 통과)

⚠️ **진행 중:**
- 일부 테스트 실패 (나중에 수정 예정)
- 프론트엔드 연동 필요

## 3. 데이터 모델 설계

### 3.1 Prisma 스키마 변경

```prisma
// Note: SQLite doesn't support enums, using String with validation

model Restaurant {
  // ...existing fields...
  
  storeType       String   @default("restaurant") @map("store_type")
  brandName       String?  @map("brand_name")
  operatingHours24 Boolean? @default(false) @map("operating_hours_24")
  
  // ...existing fields...
}

model Menu {
  // ...existing fields...
  
  requiresAgeVerification Boolean  @default(false) @map("requires_age_verification")
  ageRestriction          String   @default("none") @map("age_restriction")
  category                String?
  barcode                 String?
  stock                   Int?
  
  // ...existing fields...
}

model AgeVerification {
  id          String   @id @default(uuid())
  userId      String   @map("user_id")
  user        User     @relation(fields: [userId], references: [id])
  
  verifiedAt  DateTime @default(now()) @map("verified_at")
  expiresAt   DateTime @map("expires_at")
  
  method      String   @default("phone") @map("method")
  phoneNumber String   @map("phone_number")
  isVerified  Boolean  @default(true) @map("is_verified")
  
  createdAt   DateTime @default(now()) @map("created_at")
  
  @@index([userId])
  @@index([expiresAt])
  @@map("age_verifications")
}
```

### 3.2 주요 모델 설명

#### StoreType
- `restaurant`: 기존 식당 (기본값)
- `convenience_store`: 편의점

#### AgeRestriction
- `none`: 연령 제한 없음
- `teen`: 청소년 이용불가 (19세 미만)
- `adult`: 성인 전용 (주류 등, 19세 이상)

#### AgeVerification
- 사용자별 성인 인증 기록
- 인증 만료 시간 관리 (기본 24시간)
- 전화번호 기반 인증

## 5. 비즈니스 로직

### 5.1 편의점 생성 플로우

```
1. 어드민이 편의점 정보 입력
   - storeType: 'convenience_store'
   - name, address, coordinates
   - brandName (선택)
   - operatingHours24 (선택)

2. 시스템 검증
   - 필수 필드 확인
   - 좌표 유효성 확인

3. 편의점 생성
   - Restaurant 레코드 생성
   - storeType = 'convenience_store'

4. 응답 반환
```

### 5.2 성인 인증 플로우

```
1. 사용자가 성인 인증 요청
   - 전화번호 입력

2. SMS 인증번호 발송
   - 6자리 인증번호
   - 3분 유효

3. 사용자가 인증번호 입력

4. 시스템 검증
   - 인증번호 일치 확인
   - 만료 시간 확인

5. 성인 인증 완료
   - AgeVerification 레코드 생성
   - 24시간 유효

6. 인증 토큰 발급
```

### 5.3 성인 인증 필요 상품 주문 플로우

```
1. 사용자가 장바구니에 상품 추가
   - 성인 인증 필요 상품 포함 여부 확인

2. 결제 전 검증
   - 성인 인증 필요 상품이 있는 경우
   - 유효한 성인 인증 확인
   - 미인증 시 성인 인증 페이지로 안내

3. 주문 생성
   - OrderItem에 verifiedAt 기록
   - requiresVerification = true

4. 정상 주문 처리
```

### 5.4 주류/음식 비율 검증 플로우

```
1. 사용자가 장바구니에서 '주문하기' 클릭

2. 주문 사전 검증 API 호출 (POST /api/v1/orders/validate)
   - items: [{ menuId, quantity }]

3. 시스템 검증
   - 주류 카테고리(category === '주류') 또는 성인제한(ageRestriction === 'adult') 상품 식별
   - 주류 총액 계산
   - 음식 총액 계산
   - 비율 검증: 주류 총액 > 음식 총액인 경우

4. 검증 결과 반환
   - isValid: true/false
   - foodAmount: 음식 총액
   - alcoholAmount: 주류 총액
   - hasAlcoholViolation: 위반 여부
   - warning: 경고 메시지 (위반 시)

5. 프론트엔드 처리
   - isValid === true: 주문 진행
   - isValid === false: 경고 팝업 표시
     "관계 법령의 규정에 따라 배달 주류 구매 총액은 음식 구매액보다 적어야 합니다.
      주류 주문을 줄이거나 음식 주문액을 늘려주세요."
   - '확인' 클릭: 장바구니 수정 화면으로 이동

6. 사용자가 장바구니 수정 후 다시 검증
```

#### 검증 규칙
| 구분 | 조건 | 결과 |
|------|------|------|
| 주류만 주문 | alcoholAmount > 0, foodAmount = 0 | 통과 |
| 음식만 주문 | alcoholAmount = 0, foodAmount > 0 | 통과 |
| 주류 < 음식 | alcoholAmount < foodAmount | 통과 |
| 주류 = 음식 | alcoholAmount = foodAmount | 통과 |
| 주류 > 음식 | alcoholAmount > foodAmount | **실패** |

## 6. 구현된 코드 요약

### 6.1 StoreService
- createStore: 편의점/식당 생성
- getStores: 상점 목록 조회 (필터링 지원)
- getStoreById: 상점 상세 조회
- updateStore: 상점 정보 수정
- deleteStore: 상점 삭제
- createProduct: 상품 등록
- getProducts: 상품 목록 조회 (카테고리/성인인증 필터링)
- updateProduct: 상품 수정
- updateStock: 재고 관리 (증가/감소/설정)
- deleteProduct: 상품 삭제

### 6.2 AgeVerificationService
- requestVerification: 성인 인증 요청 (SMS 발송)
- verify: 인증번호 확인
- getStatus: 인증 상태 조회
- isVerified: 인증 여부 확인
- verifyForOrder: 주문 시 인증 확인

### 6.3 Admin Routes (확장)
- POST /stores: 편의점 생성
- GET /stores: 편의점 목록 조회
- GET /stores/:id: 편의점 상세 조회
- PUT /stores/:id: 편의점 정보 수정
- DELETE /stores/:id: 편의점 삭제
- POST /stores/:storeId/products: 상품 등록
- GET /stores/:storeId/products: 상품 목록 조회
- PUT /products/:id: 상품 수정
- PATCH /products/:id/stock: 재고 수정
- DELETE /products/:id: 상품 삭제

### 6.4 OrderService (확장)
- ageVerificationId 필드 추가
- 성인 인증 검증 로직
- 재고 검증 로직 (품절/재고부족)
- 배달 거리 검증 로직
- **validateOrderItems: 주류/음식 비율 검증 로직**
