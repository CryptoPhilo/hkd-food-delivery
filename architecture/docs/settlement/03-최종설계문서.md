## 3. 실제 데이터베이스 스키마

### 3.1 Settlement 모델 (구현 완료)

```prisma
model Settlement {
  id              String   @id @default(uuid())
  driverId        String   @map("driver_id")
  driver          Driver   @relation(fields: [driverId], references: [id])
  
  period          String   @map("period")           // "YYYY-MM"
  startDate       DateTime @map("start_date")
  endDate         DateTime @map("end_date")
  
  totalDeliveries Int      @map("total_deliveries")
  totalDeliveryFee Int     @map("total_delivery_fee")
  
  serviceFeeRate  Float    @map("service_fee_rate")  // 0.10 (10%)
  serviceFee      Int      @map("service_fee")
  taxRate         Float    @map("tax_rate")          // 0.033 (3.3%)
  tax             Int      @map("tax")
  
  bonus           Int      @default(0) @map("bonus")
  penalty         Int      @default(0) @map("penalty")
  
  grossAmount     Int      @map("gross_amount")
  netAmount       Int      @map("net_amount")
  
  status          String   @default("calculated")
  // calculated → approved → paid
  
  paidAt          DateTime? @map("paid_at")
  paidAmount      Int?      @map("paid_amount")
  
  notes           String?
  approvedBy      String?   @map("approved_by")
  approvedAt      DateTime? @map("approved_at")
  
  items           SettlementItem[]
  
  createdAt       DateTime @default(now()) @map("created_at")
  updatedAt       DateTime @updatedAt @map("updated_at")
  
  @@unique([driverId, period])
  @@index([driverId])
  @@index([period])
  @@index([status])
  @@map("settlements")
}
```

## 5. 실제 구현된 비즈니스 로직

### 5.1 정산 생성 로직 (SettlementService.ts)

```typescript
async generateMonthlySettlement(
  period: string,
  config: Partial<SettlementConfig> = {}
): Promise<GenerateResult> {
  const finalConfig = { ...this.defaultConfig, ...config };
  
  // 1. 기간 검증
  const periodRegex = /^\d{4}-(0[1-9]|1[0-2])$/;
  if (!periodRegex.test(period)) {
    throw new Error('기간 형식이 올바르지 않습니다 (YYYY-MM)');
  }
  
  // 2. 기간 계산
  const [year, month] = period.split('-').map(Number);
  const startDate = new Date(year, month - 1, 1);
  const endDate = new Date(year, month, 0, 23, 59, 59, 999);
  
  // 3. 배달 완료 주문이 있는 배달원 조회
  const drivers = await prisma.driver.findMany({
    where: {
      orders: {
        some: {
          status: 'completed',
          deliveredAt: { gte: startDate, lte: endDate }
        }
      }
    }
  });
  
  // 4. 배달원별 정산 생성
  for (const driver of drivers) {
    // 중복 체크
    const existing = await prisma.settlement.findUnique({
      where: { driverId_period: { driverId: driver.id, period } }
    });
    if (existing) continue;
    
    // 배달 실적 조회
    const orders = await prisma.order.findMany({
      where: { driverId: driver.id, status: 'completed', deliveredAt: { gte: startDate, lte: endDate } }
    });
    
    // 금액 계산
    const totalDeliveries = orders.length;
    const totalDeliveryFee = orders.reduce((sum, o) => sum + o.deliveryFee, 0);
    const serviceFee = Math.floor(totalDeliveryFee * finalConfig.serviceFeeRate);
    const tax = Math.floor((totalDeliveryFee - serviceFee) * finalConfig.taxRate);
    const netAmount = totalDeliveryFee - serviceFee - tax;
    
    // 정산 생성
    const settlement = await prisma.settlement.create({ data: { ... } });
    
    // 정산 상세 생성
    for (const order of orders) {
      await prisma.settlementItem.create({ data: { ... } });
    }
  }
}
```

### 5.2 정산 승인 로직

```typescript
async approveSettlement(settlementId: string, adminId: string, notes?: string) {
  const settlement = await prisma.settlement.findUnique({ where: { id: settlementId } });
  
  if (!settlement) throw new Error('정산을 찾을 수 없습니다');
  if (settlement.status !== 'calculated') {
    throw new Error('계산 완료 상태의 정산만 승인할 수 있습니다');
  }
  
  return prisma.settlement.update({
    where: { id: settlementId },
    data: { status: 'approved', approvedBy: adminId, approvedAt: new Date(), notes }
  });
}
```

### 5.3 지급 완료 로직

```typescript
async markAsPaid(settlementId: string, paidAmount: number, paidAt: Date, notes?: string) {
  const settlement = await prisma.settlement.findUnique({ where: { id: settlementId } });
  
  if (!settlement) throw new Error('정산을 찾을 수 없습니다');
  if (settlement.status !== 'approved') {
    throw new Error('승인된 정산만 지급 처리할 수 있습니다');
  }
  if (paidAmount <= 0) throw new Error('지급액은 0보다 커야 합니다');
  
  return prisma.settlement.update({
    where: { id: settlementId },
    data: { status: 'paid', paidAmount, paidAt, notes }
  });
}
```

## 7. 실제 금액 계산 예시

### 7.1 테스트 데이터 기반 계산

**입력:**
- 배달 완료: 150건
- 총 배달비: 450,000원 (건당 평균 3,000원)

**계산:**
- 서비스 수수료 (10%): 45,000원
- 세금 (3.3%): (450,000 - 45,000) × 0.033 = 13,365원
- 실지급액: 450,000 - 45,000 - 13,365 = 391,635원

**출력:**
```json
{
  "totalDeliveries": 150,
  "totalDeliveryFee": 450000,
  "serviceFee": 45000,
  "tax": 13365,
  "netAmount": 391635
}
```

## 8. 알려진 이슈 및 해결 사항

### 8.1 해결된 이슈

| 이슈 | 설명 | 해결 방법 |
|------|------|-----------|
| Prisma 타입 불일치 | OrderStatus 타입 에러 | 타입 제거 또는 any 사용 |
| NaN 계산 오류 | config 전달 시 undefined 값 | config 유효성 검증 추가 |
| DB 테이블 없음 | settlement_items 테이블 누락 | prisma db push 실행 |

### 8.2 현재 제한사항

1. **수수료율 고정**: 기본 10%, 3.3% (변경 가능하지만 기본값)
2. **일별 정산 미지원**: 현재 월별 정산만 지원
3. **자동 지급 미구현**: 수동 지급 처리 필요
4. **알림 미구현**: 정산 생성/승인/지급 시 알림 없음
