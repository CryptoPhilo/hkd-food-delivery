## 우선순위 정의

- **P0 (Critical)**: 핵심 기능, 사용 불가능하면 안됨, 즉시 해결 필요
- **P1 (High)**: 중요 기능, 가능한 한 빨리 구현/해결
- **P2 (Medium)**: 유용한 기능, 계획된 릴리스에 포함
- **P3 (Low)**: 개선 사항, 나중에 구현 가능

---

## 자동화 상태

- **Y (Automated)**: 자동화된 테스트가 존재하고 CI/CD에 통합됨
- **N (Manual)**: 수동 테스트 필요 또는 자동화 예정

---

## 구현 상태

- **Implemented**: 테스트 코드 작성 완료, CI 통과 가능
- **Pending**: 테스트 코드 작성 예정
- **In Progress**: 현재 작업 중
- **Backlog**: 계획 단계, 아직 시작 안함
- **Blocked**: 의존성 문제로 진행 불가
